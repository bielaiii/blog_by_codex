# 为什么像reduce， scan这样的并行算法可以工作？

> 只要运算结果满足集合律，tree reduction / parallel scan 这类算法就都能成立。

```cpp
(a + b) + c == a + (b + c)
```
你可以随便添加括号，改变计算顺序，都不影响最终的结果

## 1.reduction为什么可以像树一样合并


原始的串行求和等价于

```cpp
((((a0 + a1) + a2) + a3) + a4) + ...
```

tree reduction改造完等价于

```cpp
(a0 + a1) + (a2 + a3) + (a4 + a5) + ...
```

再下一轮
```cpp
((a0 + a1) + (a2 + a3)) + ((a4 + a5) + (a6 + a7))
```

看起来结构变了，但是加法有结合律，所有这些形式只是改变了括号的组织形式
```cpp
a0 + a1 + a2 + a3 + a4 + ...
```

tree reduction所做的事
> 每一轮把很多小区间的和合并成更大的区间和。

比如有8个元素
```cpp
第 0 层： [a0] [a1] [a2] [a3] [a4] [a5] [a6] [a7]

第 1 层： [a0+a1] [a2+a3] [a4+a5] [a6+a7]

第 2 层： [a0+a1+a2+a3] [a4+a5+a6+a7]

第 3 层： [a0+a1+a2+a3+a4+a5+a6+a7]
```

它的循环不变量是
> 第k轮结束后，每个活跃区间保存的是长度为2^k的连续长度的区间和


## 为什么scan也有类似doubling的效果

```cpp
output[0] = a0
output[1] = a0 + a1
output[2] = a0 + a1 + a2
output[3] = a0 + a1 + a2 + a3
....
```

Hillis-Steele scan 的思路是：
```cpp
offset = 1, 2, 4, 8, ...
scratch[i] += scratch[i - offset]
```
它的循环不变量是
> 第k轮结束后，scratch[i]保存最长为2^{k+1}长度的连续区间和


## 为什么拆数组和构造数组都能成立

拆数组和构造数组本质是同一件事的两个方向

reduction 是从叶子往根走
小区间 -> 大区间 -> 更大区间 -> 总和

例如：
```text
[a0] [a1] [a2] [a3]
  \   /     \   /
 [a0+a1]   [a2+a3]
      \     /
 [a0+a1+a2+a3]
```
它是在构造一棵合并树。
```text
scan 是让每个节点找到自己需要的左边区间
scratch[i] 一开始只知道 a[i]
然后知道左边 1 个
然后知道左边 2 个
然后知道左边 4 个
...
```

例如 i = 7：

```text
初始： a7

offset = 1：
a6 + a7

offset = 2：
a4 + a5 + a6 + a7

offset = 4：
a0 + a1 + a2 + a3 + a4 + a5 + a6 + a7
```
它是在给每个位置构造自己的“前缀合并树”。

4. 最根本的证明：结合律 + 区间不变量

可以把这类算法的证明归结成两句话：

第一，所有中间值都表示某个连续区间的和。

第二，每一轮合并的两个区间都是相邻且不重叠的，所以合并后仍然是一个更大的连续区间。

比如：
```text
[a0 a1 a2 a3] + [a4 a5 a6 a7]
```
合并后就是：
```text
[a0 a1 a2 a3 a4 a5 a6 a7]
```
只要这个性质一直保持，最后结果一定正确。

5. 所以它不是“加法专属”，而是更一般的东西

只要运算满足结合律，都可以做类似的 reduction / scan。

可以用的：
```text
sum:      a + b
product:  a * b
min:      min(a, b)
max:      max(a, b)
and/or:   a && b, a || b
xor:      a ^ b
```

---

所以，**数学上保证任何N叉树操作都可以**，工程实现一般用最方便的**二叉树**操作

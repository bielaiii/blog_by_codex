# 给持续写作保留一点轻量流程

很多博客做不下去，不是因为没有内容，而是因为每次发文的动作太重。

## 一个简单流程

1. 在 `posts/` 下新增一个 Markdown 文件
2. 在 `app.js` 的文章数组里加入标题、日期、摘要和标签
3. 刷新页面检查展示效果
4. 提交到 GitHub 仓库并发布

## 适合写什么

- 项目记录
- 技术方案复盘
- 工具使用经验
- 偶尔的非技术随笔

## 关于风格

我更倾向于把视觉重点放在这些位置：

- 首页第一屏要有辨识度
- 列表与正文层级要明确
- 色彩要温和，但不能太“安全”

博客不一定要花哨，但至少应该让人看得出它经过了设计。
